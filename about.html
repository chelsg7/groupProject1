About our Project

TRAIL-E finds weather and outdoor recreational trails for a specified location - either the user’s location or one they may enter. TRAIL-E also displays an area for the user to leave feedback about different trails.

TRAIL-E is designed with the user experience in mind. It brings together a variety of unique functionalities to allow the user to save time and spend more time on the trails!

  • Must meet good quality coding standards (indentation, scoping, naming)
  • Must NOT use alerts, confirms, or prompts (look into modals!)

About Bulma

Bulma is an open source CSS Framework that uses flexbox to create responsive, mobile-first design elements. It allows simple styling based on columns and tile elements.

TRAIL-E uses Bulma for page design. The internal page elements are styles as Bulma tile elements. These elements can contain about anything and are responsive to page size. The navbar, tables and footer are both Bulma styled as well.

The biggest difficulty with Bulma was learning the new required classes for styling. I also ran into some errors regarding spacing / alignment of some elements within their parent elements. Furthermore, while I found the Bulma class names to be informative and helpful, they also tend to be wordy instead of concise like some other frameworks.

About Google Maps API

The Google Maps API is unlike any other API we have seen so far. It is such a large and inclusive library that it requires a variety of special formatting to work properly.

The Google Maps API was used to display a map within our webpage. This includes the addition of pins to the map to display trail locations.

Due to these special requirements, we dealt with a number of console errors not specifically related to our code. These errors included CORS errors and errors with the Maps API communicating with the TrailAPI. The Map display cannot show markers until the TrailsAPI call comes back.

About Weather Underground API

The Weather Underground API uses two different URLs to pull information for our webpage. One URL is used for the current weather condition, and another is used for the forecast. The AJAX response from these URLs returns different parameters regarding both the current weather and the upcoming forecasted weather.

TRAIL-E displays icons and other parameters about the present condition to quickly inform the user of the current weather. The area showing upcoming forecast contains three phases (day-night-next day or night-nextday-next night).

The main difficulty was getting the city information to AJAX call properly, because it requires different formatting for The United States versus other countries. After the AJAX call was working, it was hard to decide which information was needed for TRAIL-E.

About TrailAPI

The TrailAPI contains information for tens of thousands of outdoor recreation locations including hiking and mountain biking trails,campgrounds, ski resorts, ATV trails, and more.

The API presented some new working challenges, one of them being that the API key was not passed within the URL, but rather in a separate header key within the AJAX call.

This API, while impressively robust in trail informational content, did give us some technical issues with its sluggish response times.
